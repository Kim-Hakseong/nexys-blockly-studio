# nexys-sdk

Runtime SDK for the Python code generated by **Nexys Blockly Studio**.

The visual editor emits `import nexys` programs. This package makes those
programs *actually run* — on your laptop (simulation) or on real NI hardware.

```python
import nexys
nexys.init(target="sim")          # or target="ni-pxie", backend="nidaqmx", device="PXI1Slot2"
nexys.tdms_open("run.tdms")

def loop_0():
    nexys.channels.do_write("DO0", "HIGH")
    v = nexys.channels.ai_read("AI0")
    nexys.output.log_tdms("bit_ai0", v)

def main():
    nexys.timing.loop_every(50, loop_0)

if __name__ == "__main__":
    main()
```

## Backends

| Backend | Select with | Needs | Status |
|---------|-------------|-------|--------|
| `sim`   | `init(target="sim")` (default) | nothing — pure stdlib | ✅ runs anywhere |
| `ni`    | `init(target="ni-pxie", backend="nidaqmx", device="PXI1Slot2")` | `nidaqmx` + NI-DAQmx runtime + NI hardware | ✅ code complete (needs hardware to exercise) |

The sim backend returns time-varying synthetic AI/sensor signals and records
AO/DO writes — the same behaviour as the in-browser simulator. The NI backend
maps logical channels to NI-DAQmx physical channels:

```
AI{n} -> {device}/ai{n}      DI{n} -> {device}/port0/line{n}
AO{n} -> {device}/ao{n}      DO{n} -> {device}/port1/line{n}
```

## TDMS output is real

`nexys.output.log_tdms(...)` writes a genuine **TDMS 2.0** binary file
(`nexys.tdms`), byte-compatible with LabVIEW / DIAdem / npTDMS — no NI software
required to produce it. Validated by round-trip against npTDMS (see `tests/`).

## Install

```bash
pip install -e sdk                 # core (sim + TDMS), pure stdlib
pip install -e "sdk[ni]"          # + real NI hardware (nidaqmx)
pip install -e "sdk[dev]"         # + pytest + npTDMS for the test suite
```

## Run the examples (no hardware)

```bash
python sdk/examples/bit_demo.py         # writes bit_demo.tdms
python sdk/examples/ni_acquire_demo.py  # writes ni_acquire.tdms
```

## Interfaces & self-verification (BIT)

Comms transports for talking to / verifying the instrument:

```python
from nexys.interfaces import UdpTransport, TcpTransport, SerialTransport
```

Each is a byte pipe (`send` / `recv` / `close`) and offers a `loopback()`
constructor (serial: `PtySerialLoopback`) that wires it back to itself — used by
the Built-In Test to verify link integrity with **no peer hardware**.

```python
import nexys
nexys.init(target="sim")              # or a real NI target
report = nexys.selftest.run()         # UDP + TCP + Serial loopback + channel BIT
print(report.summary())
assert report.passed
```

```
nexys BIT self-test
  [PASS] udp.loopback           sent 24B, recv 24B    0.06 ms
  [PASS] tcp.loopback           sent 24B, recv 24B   52.13 ms
  [PASS] serial.loopback        sent 24B, recv 24B   57.27 ms
  [PASS] channel.ai0_range      AI0=2.862 V on sim
  --> 4/4 checks passed (PASS)
```

`python sdk/examples/selftest_demo.py` exits 0 on PASS / 1 on FAIL — drop it in
CI or a pre-deploy gate. Real serial ports use pyserial (`pip install
nexys-sdk[serial]`); the loopback BIT itself is pure stdlib (pseudo-terminal).

## API surface (what the generator emits)

```
nexys.init(target, backend=None, device=None, sim_ticks=10, sim_realtime=True, max_seconds=30)
nexys.tdms_open(path) / nexys.tdms_close()
nexys.channels.{ai_read, ao_write, di_read, do_write, sensor_read}
nexys.timing.{loop_every, delay_ms, wait_until}
nexys.signal.{scale_linear, rms, lpf, in_range}
nexys.output.{log_tdms, mqtt_publish, alarm, bit_result}
```

In the sim backend a `loop_every` runs `sim_ticks` iterations then returns, so
generated programs terminate instead of spinning forever. On hardware it loops
until interrupted or `max_seconds` elapses.
